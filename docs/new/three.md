# 《实习三偷定律》

先上结论: 实习期间一定要多问,多学,多偷! 当然不是偷东西的偷,主要是偷产出,来一招 四两拨千斤, A姐做的＝我的，B哥做的也＝我的，只要我不说这些产出都是我的。

**实习最重要的就是整合,整合你在实习期间的所有的技术/知识,转化成可以加分的东西 比如项目/数据/经验 等等 一定要在你简历上展现。**

**转化率低的话,要么是你不会整合，要么就是太打杂了，一句话跑路。**

# 如何增加自己的产出：

**我总结了三偷定律：偷学，偷聊，偷需求**

![img](https://uploadfiles.nowcoder.com/images/20250722/769554698_1753195738581/D2B5CA33BD970F64A6301FA75AE2EB22)

# 1.偷学

**你所在的组负责的项目一定是为了解决一个具体的业务问题，对于大公司来说，你负责的项目可能是一整个大的架构中的其中一环。因此多翻一翻关于这个项目的立项书以及各种相关资料。**

领导让你做的不一定是重要的,但是领导想做的一定是重要的。

每次周会或者每次汇报 都要看看其他组员在干什么。

1. 一定要知道你们在做一件什么事情
2. 要知道你们做事情的目的是什么？解决什么问题？
3. 你们做的这个产品也好/项目也好,技术栈用到了什么。

这三个是你要偷学的,分别对应你简历的 项目内容/项目技术栈

偷学的目的就是拓宽自己技术栈,尝试让自己去匹配自己能解决的这个项目,往项目负责人的能力上去靠拢。

到时候说项目的时候 你能说出一些额外的点 就可以加深面试官对你的项目的认可度和真实度。

# 2.**偷聊**

那么这里的偷聊的意义要比偷学更加精进。

有时候都在讲 听君一席话 胜读十年书。

有时候项目代码 你看的一头雾水，你的正式员工朋友 都一定理的头头是道了。

这时候就要私底下 多问 多聊天,一定要跟正式员工或者组内大佬聊天，刚开始可以是聊技术 纯技术。

后期问的多了 也可以聊聊其他的话题，他们一般也不会对你设下防备。

**这里我们要的就是他们对于这个项目的整体理解度以及难点/技术深度，他们对于项目本身肯定比你自己去看代码来的快的多。**

# **3.偷需求**

但凡是有点体量的公司，员工在做一个需求的时候一定是要出技术方案的。这个技术方案会详细的讲他这个功能点的设计思路。

在偷技术方案的时候，我们要搞懂以下几点：

**深入理解他人技术方案的要点**

1.Why： 需求根源与价值： 这个需求因何而生？解决什么问题？上线后预期带来什么效益？尝试找到关联的产品需求文档（PRD），理解业务驱动力。

2.How & Why： 技术决策的深度解析： 方案为何选择了特定的技术（如技术A）？技术A在此场景下的优势是什么？主动思考：是否有其他技术（B、C...）也能实现？原作者选择A的关键原因是什么？（这需要结合方案调研背景和技术选型权衡）

3.What： 落地实现的检验： 该需求对应的代码改动（PR）是否存在？能否找到并仔细阅读实际代码？通过代码印证设计思路，理解具体实现细节。
